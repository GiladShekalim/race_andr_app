# Resource Values Reference — `res/values/` & `res/values-night/`

---

## `colors.xml` — Color Palette

### Day Mode (`res/values/colors.xml`)

| Name | Hex | Used For |
|---|---|---|
| `black` | `#FF000000` | System default |
| `white` | `#FFFFFFFF` | System default |
| `background_color` | `#424242` | Root `RelativeLayout` background (dark asphalt grey) |
| `road_color` | `#444444` | Declared but not referenced in current layouts — available for lane fill |
| `heart_color` | `#4CAF50` | Declared but not referenced — available for heart tinting |

### Night Mode (`res/values-night/colors.xml`)

| Name | Hex | Day Difference |
|---|---|---|
| `background_color` | `#1A1A1A` | Darker asphalt (day: `#424242`) |
| `road_color` | `#222222` | Darker road (day: `#444444`) |
| `heart_color` | `#81C784` | Lighter green for better contrast on dark bg (day: `#4CAF50`) |

The `background_color` difference creates the day/night visual mode. Day mode is a dark grey road; night mode is nearly black.

**Note:** `road_color` and `heart_color` are defined but not currently used in `activity_main.xml` or any drawable. They are placeholders for future use (e.g., if lane fills are added, or if heart tinting is applied via `app:tint`).

## `dimens.xml` — Dimension Tokens

```xml
<!-- Lives Indicator -->
<dimen name="heart_size">54dp</dimen>        <!-- AppCompatImageView width and height -->
<dimen name="heart_margin">12dp</dimen>      <!-- margin around each heart icon -->
<dimen name="heart_top_margin">60dp</dimen>  <!-- distance from screen top (camera cutout clearance) -->

<!-- Player and Game Objects -->
<dimen name="car_size">80dp</dimen>          <!-- declared but not used — car fills its cell via layout_weight -->
<dimen name="obstacle_size">64dp</dimen>     <!-- declared but not used — obstacles fill cells via layout_weight -->

<!-- Spacing -->
<dimen name="default_margin">16dp</dimen>    <!-- FAB margin from screen edges -->
<dimen name="car_bottom_margin">100dp</dimen> <!-- declared but not used -->
```

`car_size`, `obstacle_size`, and `car_bottom_margin` are defined but unused in the current layout. The grid uses `layout_weight` for all sizing, so explicit dp values for game objects aren't needed. These are likely leftover from an earlier design iteration or reserved for future use.

**`heart_top_margin = 60dp`** is the key display-cutout accommodation. Modern Android devices with punch-hole cameras (like the Pixel 9) have the camera in the top portion. 60dp margin ensures the hearts row clears that area.

## `strings.xml`

```xml
<string name="app_name">avoided_race_app</string>
<string name="money_lost">MONEY LOST!</string>
<string name="bankrupt">BANKRUPT!</string>
```

Only 3 strings. The `money_lost` and `bankrupt` strings are used in `handleCollision()` and `resetGame()` to swap the overlay text:
```kotlin
(main_LBL_money_lost as android.widget.TextView).text = getString(R.string.money_lost)
// or
(main_LBL_money_lost as android.widget.TextView).text = getString(R.string.bankrupt)
```

The LEFT/RIGHT button labels are hardcoded in XML (`android:text="LEFT"` / `android:text="RIGHT"`) — they should ideally be string resources.

## `themes.xml` — App Theme

### Day Mode (`res/values/themes.xml`)
```xml
<style name="Base.Theme.Avoided_race_app" parent="Theme.Material3.DayNight.NoActionBar">
    <!-- Customize your light theme here. -->
</style>
<style name="Theme.Avoided_race_app" parent="Base.Theme.Avoided_race_app" />
```

### Night Mode (`res/values-night/themes.xml`)
```xml
<style name="Base.Theme.Avoided_race_app" parent="Theme.Material3.DayNight.NoActionBar">
    <!-- Customize your dark theme here. -->
</style>
```

Both extend `Theme.Material3.DayNight.NoActionBar`:
- **Material 3** — latest Material Design system (used for `ExtendedFloatingActionButton`, `MaterialTextView`)
- **DayNight** — automatically selects day/night resources based on system setting
- **NoActionBar** — removes the top app bar (game takes full screen)

No color overrides are applied — the theme uses Material 3 defaults for button styling, with the game's own `background_color` as the screen background.

## `xml/backup_rules.xml` & `xml/data_extraction_rules.xml`

Standard Android boilerplate generated by Android Studio. Controls what data is backed up to Google Drive (backup_rules.xml) and what can be extracted in Android data backup scenarios (data_extraction_rules.xml). Not game-relevant — no persistent data to back up currently.

## Adding New Resource Values

### New color for future score display:
```xml
<!-- in values/colors.xml -->
<color name="score_text_color">#FFFFFF</color>

<!-- in values-night/colors.xml -->
<color name="score_text_color">#E0E0E0</color>
```

### New string for pause button:
```xml
<string name="pause">PAUSE</string>
<string name="resume">RESUME</string>
```

### New dimensions for score text:
```xml
<dimen name="score_text_size">28sp</dimen>
<dimen name="score_top_margin">16dp</dimen>
```

### If adding a start screen with a title:
```xml
<string name="game_title">Avoided Race</string>
<string name="tap_to_start">Tap to Start</string>
<string name="high_score_label">Best: %d</string>  <!-- %d for integer formatting -->
```
